# UI 重设计方案对比

> 请阅读后告诉我选择哪个方案，或者有什么综合意见。

---

## 公共设计语言（所有方案共用）

- **风格**：极简白底，参考 Linear / Vercel Dashboard
- **颜色**：背景 #FFFFFF，文字 #111827，主色 Indigo #4F46E5，边框 #E5E7EB（极细）
- **卡片**：无阴影，1px 细边框，圆角 12px
- **字体**：系统无衬线，标题 font-semibold，正文 font-normal
- **暗色模式**：TopBar 右侧加一个日/夜切换按钮，整体反色
- **动效**：所有 transition 统一 150ms ease，禁止过度动效

---

## 方案 A：编辑器折叠 + 视频主导

**核心理念**：视频是产品的主角，编辑器是辅助工具。右侧视频预览更宽更高，左侧编辑器用折叠面板压缩层级。

```
┌─────────────────────────────────────────────────────────────────┐
│  ☁ Problem Explainer Studio  │ 八股/语法 │ 算法图解 │ 批量 │ ⚙ 🌙 │
├────────────────────────┬────────────────────────────────────────┤
│                        │                                        │
│  ▼ AI 智能解析（展开）  │   ┌────────────────────────────────┐  │
│  ┌──────────────────┐  │   │                                │  │
│  │  粘贴题目文本...  │  │   │                                │  │
│  └──────────────────┘  │   │         视频预览                │  │
│  模型: [DeepSeek V4 ▾] │   │       (16:9，更宽更高)          │  │
│  [  一键解析题目  ]    │   │                                │  │
│                        │   └────────────────────────────────┘  │
│  ▶ 手动编辑（折叠）    │                                        │
│  ▶ 图解配置（折叠）    │   时长: 45s  ·  帧数: 1350  ·  ✅配音  │
│                        │                                        │
│                        │   ┌──────────────┐ ┌──────────────┐   │
│                        │   │ 🎙 生成配音预览 │ │ ⬇ 导出 MP4  │   │
│                        │   └──────────────┘ └──────────────┘   │
│                        │                                        │
│       (42%)            │              (58%)                     │
└────────────────────────┴────────────────────────────────────────┘
```

**优点**
- 视频预览区域明显更大，导出按钮突出
- AI 解析是默认展开项，强调"一键生成"的核心流程
- 手动编辑折叠起来，不干扰主流程
- 操作按钮（生成配音、导出）和视频在同侧，符合"看→操作"习惯

**缺点**
- 手动编辑内容多时，左栏滚动和右栏 sticky 视频需要仔细处理
- 折叠/展开需要额外的交互学习成本

## 方案 B：三栏流水线

**核心理念**：工作流线性化。输入 → 结构化结果 → 预览导出，三步各占一栏，视觉上很清楚"做到哪一步了"。

```
┌─────────────────────────────────────────────────────────────────┐
│  ☁ Problem Explainer Studio  │ 八股/语法 │ 算法图解 │ 批量 │ ⚙ 🌙 │
├──────────────┬───────────────────────────┬───────────────────────┤
│              │                           │                       │
│  Step 1      │  Step 2                   │  Step 3               │
│  原始输入    │  AI 解析结果              │  预览 & 导出          │
│  ──────────  │  ─────────────────────    │  ─────────────────    │
│              │                           │                       │
│  粘贴题目:   │  题目标题: ___________    │  ┌─────────────────┐  │
│  ┌────────┐  │                           │  │                 │  │
│  │        │  │  核心考点:                │  │   视频预览       │  │
│  │        │  │  • 考点一                 │  │   (16:9)        │  │
│  └────────┘  │  • 考点二                 │  │                 │  │
│              │  • 考点三                 │  └─────────────────┘  │
│  AI 模型:    │                           │                       │
│  [▾ 选择]   │  讲解文案:                │  时长: 45s            │
│              │  ┌─────────────────────┐  │                       │
│ [一键解析→] │  │ 这道题考察了...     │  │  [🎙 生成配音预览]   │
│              │  └─────────────────────┘  │  [⬇  导出 MP4  ]     │
│  (25%)       │         (40%)             │       (35%)           │
└──────────────┴───────────────────────────┴───────────────────────┘
```

**优点**
- 工作流程极其清晰，用户知道自己在哪一步
- 三栏等宽，视觉平衡
- 每栏职责单一，不会让用户迷失

**缺点**
- 屏幕宽度要求高（1440px 以上才舒适，1280px 会很挤）
- 视频预览宽度受限（最窄的一栏），观感上偏小
- 三栏中间的"AI 解析结果"区域，用户只能看不能太多编辑，如果 AI 返回结果有问题，编辑体验较差

## 方案 C：全屏编辑 + 侧拉预览

**核心理念**：专注模式。编辑器全宽，写完了再拉出来看视频。类似 Figma 右侧属性面板的交互方式。

```
【编辑模式（默认）】
┌─────────────────────────────────────────────────────────────────┐
│  ☁ Problem Explainer Studio  │ 八股/语法 │ 算法图解 │ 批量 │ ⚙ 🌙 │ [预览 ▶]│
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ✨ AI 智能一键解析                              RECOMMENDED     │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  粘贴题目文本...                                          │   │
│  └──────────────────────────────────────────────────────────┘   │
│  模型: [DeepSeek V4 Flash ▾]          [     一键解析题目     ]   │
│                                                                  │
│  ─────────────────── Manual Edit ───────────────────────────    │
│  题目标题: _________________________                              │
│  核心考点: • _______________________ [+ 添加]                    │
│  讲解文案: ___________________________________________________    │
│                                                                  │
│  [  🎙 生成配音预览  ]   [  ⬇ 导出 MP4  ]                       │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

【点击"预览 ▶"后，右侧面板滑入】
┌────────────────────────────────┬────────────────────────────────┐
│  编辑器（自动收窄）             │  ◀ 关闭   视频预览面板          │
│                                │                                 │
│  ✨ AI 解析（紧凑版）           │  ┌─────────────────────────┐   │
│  [粘贴框][模型][解析]          │  │                         │   │
│                                │  │      视频预览 (16:9)    │   │
│  ▶ 手动编辑（折叠）            │  │                         │   │
│                                │  └─────────────────────────┘   │
│                                │  时长: 45s · 帧: 1350           │
│                                │                                 │
│                                │  [ 🎙 生成配音 ] [ ⬇ 导出 ]    │
└────────────────────────────────┴────────────────────────────────┘
```

**优点**
- 编辑时全屏专注，不被视频预览分心
- 视频预览面板可大可小（可调节）
- 对窄屏友好，移动端也能正常使用

**缺点**
- 需要额外点击"预览"按钮才能看到视频，增加了一步操作
- "实时预览"的特性被弱化了（不能边改边看）
- 比其他方案多一个交互状态，开发复杂度稍高

## 我的综合推荐

基于这个工具的核心用途（快速生成视频），我推荐 **方案 A（折叠 + 视频主导）**，理由：

1. **视频生成工具，视频就该是主角**：右侧更大的预览区直接体现产品价值
2. **保留实时预览**：用户改一下马上能看到效果，不需要额外点击
3. **折叠面板解决左侧内容多的问题**：平时只显示 AI 解析框，手动编辑折叠起来，界面干净
4. **操作按钮在视频旁边**：生成配音→看视频→导出，三个动作在同一区域完成，符合视觉动线

如果你更看重**工作流清晰度**，选 B；如果你更看重**专注写作**，选 C。

## 补充：暗色模式实现方式

无论选哪个方案，暗色模式都通过 Tailwind 的 `dark:` 类 + `<html>` 上的 `class="dark"` 实现，切换逻辑存 `localStorage`。不影响布局结构，可以独立完成。
